# Документация Проекта OSI

## Общее описание

OSI - это веб-приложение, реализующее операционную систему с графическим интерфейсом (GUI) в браузере. Оно состоит из различных компонентов и приложений, имитирующих работу классической операционной системы.

## Структура проекта

### Основные директории

- **src/** - Исходный код приложения
- **docs/** - Сгенерированная документация и ресурсы для продакшена
- **public/** - Публичные ресурсы (HTML, favicon)
- **DevelopDocumentations/** - Документация разработки

### Структура src/

#### Каталоги

1. **src/_stores/** - Хранилища данных для Pinia
2. **src/apps/** - Приложения системы
3. **src/assets/** - Асеты (изображения, иконки)
4. **src/components/os/** - Компоненты операционной системы
5. **src/config/** - Конфигурационные файлы
6. **src/core/** - Ядро приложения
7. **src/styles/** - Стили

#### Основные файлы

- `src/App.vue` - Главный компонент приложения
- `src/main.js` - Точка входа в приложение
- `src/config/applications.js` - Конфигурация приложений
- `src/config/os.js` - Конфигурация операционной системы

## Конфигурации

### applications.js
Файл отвечает за конфигурацию приложений, их имена, пути, иконки и другие параметры. Каждое приложение описывается через класс OSIApp с параметрами:
- id: уникальный ID приложения
- name: имя приложения
- label: отображаемое название
- category: категория (system, utilities, games)
- iconclass: CSS классы для иконки
- defWidth/defHeight: стандартные размеры окна
- resizable/canMinimize: параметры управления окном
- showOnDesktop/showInStartMenu: где отображать приложение

### os.js
Конфигурационный файл операционной системы с основными настройками:
- Имя системы (OSI)
- Версия
- Дата сборки
- Переводы названий версий на русский и английский

## Хранилища данных

### depr_apps.store.js
Хранилище приложений использует Pinia для управления состоянием окон. Сохраняет в localStorage информацию о состоянии окон (размер, позиция, свернутость) с автоматическим сохранением через timeout.

## Помощь в разработке

Проект использует Vue.js 3 с Composition API и Pinia для управления состоянием. Всё приложение построено как модульная система, где каждый компонент может быть независимо разработан и протестирован.

## Структура приложений

Все приложения находятся в директории `src/apps/` и организованы в следующую структуру:
- `system/` - системные приложения
- `utilities/` - служебные приложения
- `games/` - игровые приложения
- `learn/` - обучающие материалы

Каждое приложение имеет стандартную структуру:
- `.vue` файл с компонентом интерфейса
- `template.html` - шаблон HTML
- `style.css` - стили приложения
- `lang.js` - файл локализации (если требуется)

## Как запустить проект

Для запуска проекта необходимо выполнить следующие шаги:

1. Установить зависимости: `npm install`
2. Запустить сервер разработки: `npm run serve`

## Документация для разработчика

### Правила написания приложений

В проекте OSI все приложения создаются по единообразному шаблону:

1. **Структура директории приложения**:
   - Все приложения находятся в `src/apps/system/` или других поддиректориях
   - Каждое приложение должно иметь следующую структуру:
     ```
     apps/system/OSINewApp/
       ├── OSINewApp.vue          # Основной компонент
       ├── template.html          # HTML-шаблон
       ├── style.css              # Стили приложения
       └── lang.js                # Файл локализации (если требуется)
     ```

2. **Формат файла конфигурации**:
   - При добавлении нового приложения в `src/config/applications.js` необходимо создать экземпляр класса `OSIApp`
   - Все параметры приложения следует указывать согласно шаблону:
     ```javascript
     new OSIApp({
         id: 'unique-app-id',
         name: 'AppName',
         label: 'Отображаемое имя',
         category: 'system', // или utilities, games и т.д.
         iconclass: 'bi-icon-name text-ico-tech',  // CSS класс для Bootstrap Icons
         defWidth: 800,
         defHeight: 450,
         resizable: true,           // Можно ли изменять размеры окна
         canMinimize: true,         // Можно ли сворачивать окно
         showOnDesktop: true,       // Отображать на рабочем столе
         showInStartMenu: true,     // Показывать в меню "Пуск"
         deskContextMenu: true      // Показывать в контекстном меню рабочего стола
     })
     ```

3. **Структура Vue-компонента**:
   - Используется Composition API с `setup()` функцией или Options API
   - Компоненты должны быть самодостаточными
   - Все HTML содержимое должно находиться в `template.html`
   - Стили реализуются через `style.css`
   - Для иконок используются Bootstrap Icons с соответствующими CSS классами

4. **Советы по разработке**:
   - При создании новых приложений следуйте уже существующим примерам
   - Используйте те же структуры файлов и названия для консистентности
   - Все новые функции должны быть интегрированы в систему управления окнами
   - При добавлении локализации используйте файл `lang.js` с нужными локалями

5. **Интеграция с системой**:
   - Для открытия приложений используется общий механизм через `SimpleWindow` компонент
   - Все приложения автоматически подключаются к меню "Пуск" и рабочему столу при конфигурации в `applications.js`
   - Пользовательские данные сохраняются через IDB хранилище

6. **Система управления окнами**:
   - Все приложения открываются внутри компонента `SimpleWindow`, который предоставляет стандартный интерфейс с заголовком, кнопками управления и контентом
   - Окна обладают параметрами: максимальное/минимальное состояние, размеры, позиционирование, z-index для порядка отображения
   - Система управления окнами позволяет перемещать, изменять размеры, сворачивать и активировать окна
   - Закрытие окон автоматически обновляет хранилище данных пользователя

7. **Поддержка состояний**:
   - Приложения могут хранить свое состояние через `localStorage` или IDB (IndexedDB)
   - Состояния окон сохраняются при закрытии и восстанавливаются при следующем открытии
   - Используется реактивное управление состоянием через Pinia store

8. **Система событий**:
   - Все коммуникации между компонентами осуществляются через центральный механизм событий
   - События могут быть как внутренними (внутри одного приложения), так и внешними (между разными приложениями)
   - Для отправки и получения событий используются методы: $emit, $on или специальные функции из ядра системы

9. **Локализация интерфейса**:
   - Приложения могут использовать файл lang.js для поддержки различных локалей
   - Используется стандартный подход с объектами переводов по ключам
   - Поддерживаемые локали: ru (русский), en (английский) и др.

10. **Интеграция с IDB хранилищем**:
    - Для работы с базой данных пользовательских данных используется IndexedDB
    - Все приложения имеют доступ к API через `src/idb/db.js`
    - Рекомендуется использовать готовые методы для сохранения/загрузки информации

## Библиотека вспомогательных функций

Проект использует централизованную библиотеку.helpers.js (`src/core/helpers.js`) для предоставления общих утилит и функциональности. Она экспортирует объект JSH с следующими категориями:

### 1. Браузерные функции
- `detectBrowser()` - определение типа и версии браузера пользователя
- `getLocalStorageUsage()` - анализ использования localStorage, включая объем данных по записям

### 2. Локализация (языковые данные)
- Системный объект `lang` с переводами для различных категорий приложений:
  - utilities (утилиты)
  - system (системные)
  - games (игры)

### 3. Системные функции
- `getImageList()` - получение списка всех изображений из директории `src/assets/wallpapers/`
- `getAvatarsList()` - получение списка аватаров из директории `src/assets/avatars/`

### Примеры подключения и использования

**Подключение библиотеки:**
```javascript
// В компоненте Vue или другом JS файле
import { JSH } from '@/core/helpers.js';
```

**Использование функций:**

1. Определение браузера:
```javascript
const browser = JSH.browser.detectBrowser();
console.log(`Браузер: ${browser.name}, Версия: ${browser.version}`);
```

2. Анализ localStorage:
```javascript
const usage = JSH.browser.getLocalStorageUsage();
if (usage) {
    console.log(`Использовано ${usage.total.kilobytes} KB из 5 MB`);
}
```

3. Получение списка изображений:
```javascript
// Для получения обоев рабочего стола
const wallpapers = JSH.system.getImageList();

// Для получения аватаров пользователей
const avatars = JSH.system.getAvatarsList();
```

4. Использование локализации (пример):
```javascript
// Получение перевода по ключу
const categoryLabel = JSH.lang.ru.cattype_system; // "Система"
```

Эта библиотека упрощает доступ к часто используемым функциям и позволяет использовать общие механизмы в разных частях приложения без дублирования кода.
